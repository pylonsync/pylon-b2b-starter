# Deploying

Four apps deploy independently because they cache and scale differently:

| App | Best fit |
|---|---|
| `apps/marketing` | Static-friendly. Vercel, Netlify, Cloudflare Pages. |
| `apps/docs` | Static-friendly. Same options as marketing. |
| `apps/dashboard` | Needs Node runtime for SSR + auth cookies. Vercel, Fly, Render. |
| `apps/backend` | Pylon framework. **Pylon Cloud (recommended)**, Fly, your own VPS. |

## Pylon Cloud (recommended for the backend)

The fastest path: push the backend to Pylon Cloud, point your
dashboard's `PYLON_BASE_URL` at the resulting `<slug>.pyln.dev`
URL, deploy the Next apps to Vercel.

```bash
# Sign up at https://cloud.pylonsync.com, then from the dashboard:
#   1. New Project → connect this repo
#   2. Set "App directory" to apps/backend
#   3. Pick a region close to your users (iad for US-East)
#   4. (Optional) attach a custom domain — every project also gets
#      <slug>.pyln.dev free
```

Pylon Cloud handles: machine provisioning + autoscaling + push-to-
deploy from main + per-project secrets + TLS + custom domains +
metrics + audit log + backups. The control plane runs the same
binary you'd run yourself, just managed.

Pricing: free hobby tier is 1 project, shared-cpu-1x, 512 MB,
autostop-on-idle. Pro tier opens up scaling, replicas, always-warm.

## Pylon backend on Fly (self-managed)

If you want Fly directly (you own the deploy, the secrets, the
scaling):

```bash
cd apps/backend
fly launch --no-deploy
fly volumes create pylon_data --size 10
# Edit fly.toml — mount /data, set DATABASE_URL=sqlite:///data/pylon.db
fly deploy
```

For Postgres, set `DATABASE_URL=postgres://...` and skip the volume — Fly's Postgres add-on or any external Postgres works.

## Vercel (for the Next apps)

```bash
# Marketing
cd apps/marketing
vercel --prod
# Set env: NEXT_PUBLIC_DASHBOARD_URL, NEXT_PUBLIC_DOCS_URL, NEXT_PUBLIC_SITE_URL

# Dashboard
cd apps/dashboard
vercel --prod
# Set env: PYLON_BASE_URL (your Pylon Cloud URL or Fly host),
#          STRIPE_SECRET_KEY, RESEND_API_KEY, …
```

Each app is a standalone Vercel project so they get their own domains, env vars, and scaling profiles.

## Self-hosted (one VPS)

For tiny deployments, run all three apps + Pylon on a single $5 VPS behind Caddy:

```caddyfile
acme.example.com {
  reverse_proxy localhost:3000
}
app.acme.example.com {
  reverse_proxy localhost:3001
}
docs.acme.example.com {
  reverse_proxy localhost:3002
}
api.acme.example.com {
  reverse_proxy localhost:4321
}
```

Use `pm2` or a systemd unit per app + `pylon dev --prod` for the backend.

## Environment variables

<AccordionGroup>
  <Accordion title="apps/marketing" icon="globe">
    - `NEXT_PUBLIC_SITE_URL` — your marketing canonical URL
    - `NEXT_PUBLIC_DASHBOARD_URL` — dashboard origin (for "Sign up" CTAs)
    - `NEXT_PUBLIC_DOCS_URL` — docs origin
  </Accordion>
  <Accordion title="apps/dashboard" icon="layout">
    - `PYLON_BASE_URL` — internal Pylon URL (server-to-server)
    - `NEXT_PUBLIC_PYLON_URL` — public Pylon URL (browser-to-server)
    - `STRIPE_SECRET_KEY`
    - `STRIPE_WEBHOOK_SECRET`
    - `NEXT_PUBLIC_STRIPE_PUBLISHABLE_KEY`
    - `RESEND_API_KEY` (if using Resend for invitation emails)
  </Accordion>
  <Accordion title="apps/docs" icon="book-open">
    - none required for `mintlify dev`. Hosting on Mintlify Cloud → push to GitHub.
  </Accordion>
</AccordionGroup>
