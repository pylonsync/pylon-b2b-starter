# Quickstart

<Steps>
  <Step title="Install the Pylon CLI">
    ```bash
    bun add -g @pylonsync/cli
    pylon --version
    ```

    The CLI runs the local Pylon dev server (`pylon dev`) and is
    invoked from the backend's `dev` script under turbo. If you have
    the framework checked out locally, you can also symlink
    `target/release/pylon` to anywhere on `$PATH`.
  </Step>

  <Step title="Clone & install">
    ```bash
    git clone https://github.com/pylonsync/pylon-b2b-starter.git
    cd pylon-b2b-starter
    bun install
    ```
  </Step>

  <Step title="Run everything">
    ```bash
    bun dev
    ```

    Turbo brings up four services in parallel:

    - **Backend** (`apps/backend`) — Pylon at `http://localhost:4321`
      with Studio at `/studio`. Schema for `User`, `Organization`,
      `OrgMember`, `Invitation`, `Subscription`, `ApiKey`, and
      `AuditEvent` is created automatically on first run.
    - **Marketing** — `http://localhost:3000`
    - **Dashboard** — `http://localhost:3001`
    - **Docs** (this site) — `http://localhost:3002`

    The dashboard proxies `/api/*` and `/studio/*` to the backend
    same-origin via Next rewrites, so the browser never makes
    cross-origin requests.
  </Step>

  <Step title="Sign up">
    Visit the dashboard, hit **Sign up**, create your first user. Onboarding walks you through:

    1. Pick a workspace name
    2. (Optional) invite teammates
    3. Pick a plan (Free / Pro / Team)

    You're now inside a fully working multi-tenant SaaS.
  </Step>
</Steps>
